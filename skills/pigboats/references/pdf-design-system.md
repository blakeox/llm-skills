# PigBoats.COM PDF Report Design System

Complete reference for generating professional PDF reports using WeasyPrint. This covers the full CSS design system, HTML component patterns, color tokens, and generation workflow.

## Setup

### WeasyPrint Virtual Environment

If `/tmp/pdfgen/` doesn't exist (it's in /tmp so may be cleared on reboot):

```bash
python3 -m venv /tmp/pdfgen && source /tmp/pdfgen/bin/activate && pip install weasyprint
```

**macOS dependency**: Requires pango — `brew install pango`

**Important**: The venv IS `/tmp/pdfgen/` itself. Activate with `source /tmp/pdfgen/bin/activate`, NOT `source venv/bin/activate`.

### Generation Pattern

Every PDF is generated by a self-contained Python script that embeds the full HTML+CSS inline:

```python
#!/usr/bin/env python3
"""Generate [report type] for [audience]."""
from weasyprint import HTML

html_content = r"""
<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
<style>
  /* Full CSS here — see Design Tokens and Component CSS below */
</style>
</head>
<body>
  <!-- Full HTML here — see Component HTML Patterns below -->
</body>
</html>
"""

output_path = "/Users/aaron/Documents/claude-code/pigboats-<topic>-<type>-<date>.pdf"
HTML(string=html_content).write_pdf(output_path)
print(f"PDF written to {output_path}")
```

**Run**: `source /tmp/pdfgen/bin/activate && python /tmp/pdfgen/generate_<name>.py`

Save scripts to `/tmp/pdfgen/generate_<name>.py` for reuse.

## Report Page Structure

### Proposal Report (Blue Theme, ~5 pages)

1. **Cover**: "Proposed Maintenance" badge, blue accent, scope stats
2. **What We Found**: Summary, green callout ("nothing changes"), table of proposed changes
3. **Phase Details**: Phase banners with risk badges, plain-English descriptions per change
4. **Test Plan**: Scoreboard showing what will be tested after each phase
5. **Bottom Line**: Deferred items, strategy box with "Preserving the Mission"

### Completion Report (Green Theme, ~5 pages)

1. **Cover**: "Mission Complete" badge, green accent, results stats
2. **What Was Done**: Summary table, green callout, backup callout
3. **What Each Change Does**: Phase banners with detail tables
4. **Test Results**: Scoreboard with all 15 tests, "All Systems Go" summary
5. **What This Means**: Win cards, strategy box, defense in depth table, submarine analogy, system status, footer

Adapt the specific details (security, speed, SEO, etc.) to match whatever work was done.
