# Response Templates

Reusable structures for common helpdesk scenarios. These are starting points, not rigid forms. Adapt to the situation.

## Standard Client Response

```
[Opening acknowledgment — 1 sentence referencing the issue]

[Findings — what we found, plain language, 1-3 sentences]

[Action taken — what we did, ownership language, 1-3 sentences]

[Next steps or Call to Action — only if needed]

[Closing — reassurance, monitoring, availability]
```

## Issue Resolved (No Action Required)

```
We looked into the [issue type] you reported on [date/timeframe].

[Root cause or finding in plain language.]

We [corrected/adjusted/updated] [specific action]. [Brief confirmation it's working.]

We'll continue to monitor this. If anything changes, let us know.
```

## Issue Under Investigation

```
We're actively investigating the [issue type] you reported.

So far, we've [specific progress — narrowed down, isolated, identified pattern].

[What happens next — timeline if known, what we're testing, who we're coordinating with.]

We'll update you as soon as we have more information.
```

## Vendor-Related Issue

```
We traced the [issue type] to [vendor/system name].

[What the vendor issue is, in plain language. No blame, just facts.]

On our end, we've [workaround or mitigation steps taken].

[Vendor coordination status if applicable.] We'll keep you updated as this progresses.
```

## Call to Action Required

```
[Acknowledgment and findings as above.]

**What we need from you:**

- [Clear step 1]
- [Clear step 2 if needed]

Once you've completed this, let us know and we'll verify everything on our end.
```

## Internal Technical Note

```
**Summary:** [1-sentence overview]

**Timeline:**
- [timestamp] — [event]
- [timestamp] — [event]

**Root Cause:** [technical detail]

**Evidence:** [quoted log lines, error codes]

**Resolution:** [what was done]

**Monitoring:** [what to watch for]
```
